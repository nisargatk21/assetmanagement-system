PROJECT NAME:
Asset Tracker

1. Understanding of Client Requirements:
The client wants a software system that helps the company track its assets. Each asset (like laptop, mobile, ID card, etc.) should be assigned to an employee. The system should clearly show which asset is currently with which employee.

2. Creative Freedoms Taken:
 product a name: Asset Tracker
 Assumed this is a web-based application
 Assumed only company admins can manage assets

3. Assumptions Made:
 Each employee has a unique employee ID
 Each asset has a unique asset ID
 One asset can be assigned to only one employee at a time
 Admin is responsible for assigning and removing assets
 Login system is required for security

4. Features List:
Employee Management
1.Add new employee
2.View all employees
3.View single employee
4.Update employee details
5.Delete employee

Asset Management
1.Add new asset
2.View all assets
3.View single asset
4.Update asset details
5.Delete asset
6.Mark asset as broken
7.Mark asset as available

Asset Assignment
1.Assign asset to employee
2.View all assignments
3.View assignment details

Asset Return
1.Return asset
2.Update asset status to available after return

Asset Handover Confirmation
1.Employee confirms asset received
2.View confirmation status

Asset Status
1.View available assets
2.View assigned assets
3.View broken assets

Asset History
1.View asset usage history
2.View employee asset history

5. Questions for Next Client Meeting:
•What types of assets should be tracked
•Should employees be able to log in
•Do you want email notifications for asset assignment
•Should asset history be maintained
•Do you need reports for assets and employees
